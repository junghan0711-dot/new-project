## 管理制度優化建議

### 1. 專案頁補齊最低管理欄位

每個 `專案-*.md` 至少補齊以下欄位，否則週會只能靠口頭記憶：

- 本月交付物
- 下一個會議/送件日期
- 業主窗口
- 本公司主責
- 目前卡點
- 最近一次更新日期

### 2. 建立每案交付清單

建議每個進行中案子新增或固定維護「交付清單」段落：

| 類別 | 檢查項目 |
|---|---|
| 合約 | 契約、報價、用印、付款節點 |
| 會議 | 議程、簡報、紀錄、待辦 |
| 執行 | 訪視、課程、工作坊、活動、諮詢 |
| 成果 | 月報、期中、期末、照片、簽到、量化表 |
| 對外 | 公文、正式提交檔、社群貼文、新聞素材 |

### 3. 週會固定節奏

建議每週會議只看 4 件事：

1. 本週要交什麼
2. 本週誰卡住
3. 哪些案子 30 天內有重大節點
4. 哪些資料還沒進專案庫

### 4. 優先改善的三個專案頁

| 優先順序 | 專案 | 為什麼 |
|---|---|---|
| 1 | 115年雲嘉南多元就業開發方案 | 已有完整進度資料，最適合做成管理示範頁 |
| 2 | 115年度新竹縣地方創生輔導案 | 資料正確性要求高，且近期正在產出專家名單與輔導資料 |
| 3 | 大溪小慈心庇護農場 | 多條工作線並行，若不拆清楚容易混在一起 |
