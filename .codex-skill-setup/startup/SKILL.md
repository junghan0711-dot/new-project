# 開工接續助手

用於新對話開始時快速恢復專案脈絡。

## 原則

- 不主動 `git pull`，只提醒使用者遠端是否有新 commit。
- 不主動修改工作筆記；開工只讀不寫。
- 工作筆記找不到時，回報並詢問 Obsidian vault 路徑。
- 摘要工作筆記，不全文貼出。

## SOP

1. 判斷工作目錄：確認 `$PWD` 是 git repo，讀 `CLAUDE.md` 或 `AGENTS.md`。
2. 找工作筆記：優先用 `CLAUDE.md` 記錄的 Obsidian 路徑；沒有路徑就詢問使用者。
3. 摘要上次進度：讀「上次做到哪」、「下一步」、「踩坑筆記」中跟當前工作直接相關的內容。
4. 檢查本地狀態：執行 `git status --short`。
5. 檢查遠端：30 分鐘內已 fetch 過就略過；否則可執行 `git fetch origin`，再檢查是否落後。
6. 回報：
   - 專案名稱
   - 上次做到哪
   - 本地 git 狀態
   - 遠端狀態
   - 建議下一步

## 不該做

- 不主動 pull。
- 不硬建工作筆記。
- 不把 `.claude/`、`.codex/`、token 或 credential 加入 git。
